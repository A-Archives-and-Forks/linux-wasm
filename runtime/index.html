Linux/Wasm

Examples: ls          watch uptime              head /proc/cpuinfo
          pwd         usleep 1234567            ps | grep kthreadd
          top         vi file.txt               find /proc -name cmdline -maxdepth 2
          mount       exec sh                   echo Hello >> world && cat world
          iostat      strings /bin/busybox      grep "Cpus_allowed_list" < /proc/self/status

The console takes over Ctrl + C etc. Depending on your platform and browser, adding Shift to the combo may work. Using Ctrl + Insert for copy and Shift + Insert for paste may also work. Right-clicking and using the context menu should also work.

This is a proof-of-concept of the Linux kernel booting in your web browser, powered by WebAssembly (Wasm). The included programs (shell and standard commands) are provided by BusyBox, backed by a musl libc implementation. The terminal emulator is provided by Xterm.js.As always, if you are unsure about how some piece of software works, take a look at the source code!

Release for Linux 7.x-era

The original release by Joel Severin was based on 6.x-era kernels. Some of the changes since that are listed below.

  • As it turns out, there exists another Wasm port from the same era written by Tom (@tombl). Whatever is currently running is the start of an effort to combine the best of both ports, but it's still work in progress and pending further review. Tom's port also has virtio and device tree support, both good features that will eventually be added here too.
  • 64-bit kernel support (optional).
  • Many small improvements and bug fixes.
